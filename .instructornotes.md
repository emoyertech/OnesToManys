# Instructor Notes

## OnesToManys Project Overview

OnesToManys is a dealership database project centered on master-detail relationships. Students build a Flask + SQLite backend and a single polished workspace for dealership and car management.

## Learning Goals

- Model a one-to-many relationship with dealerships and cars.
- Implement CRUD routes for both tables.
- Build relationship endpoints.
- Add JSON import and export support.
- Present the app as a dealership database only.

## Suggested Flow

### Foundation
- Database schema
- Seed data
- CRUD routes

### Relationships
- Dealership-to-car lookup routes
- Scoped detail retrieval

### Frontend
- Unified workspace page
- Import/export controls
- Relationship explorer

## Support Notes

- Keep terminology consistent: dealership, car, inventory, import, export.
- Avoid generic project labels in student-facing materials.
- Emphasize validation, referential integrity, and predictable JSON responses.
